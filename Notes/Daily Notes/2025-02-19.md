- 11.0 cut will come after
    - Once 11.0 is cut, develop will become **the** 11.0 branch. Any branch that was off of 11.0 Release should be treated as if it's off of develop now.
    - The 11.0 Release branch be treated as a proper release branch, meaning nothing will go in there afterwards.
    - During the time of the cut, there will a code freeze. Afterwards, business as usual: anything that happens to get merged into develop will become part of the next release.
- Future 10.X patches
    - Typically, our process is to push people to update to the new version of SeisWare, so chances of another patch are very slim, even 10.8.8 is kind of an exception.
    - In the unfortunate case of a 10.X patch, the fixes will be committed in two places: develop and a branch based on the latest version of 10.X release.
    - Depending on the changes; we'll either be able to cherrypick them into the patch branch from develop or have to implement again for the 10.X codebase. Either way, it's double testing!
- Future Minor/Major releases
    - There will be work that is meant for minor or major releases only and not 11.0. In which case, we'll create a **staging** branch for that release until it's ready to merge to develop. There will not be using the release branch like we did with 11.0. That was bad.

That's it! Is that clear to everyone? Let me know if something isn't clear, or if you have any questions. if I missed anything from the discussion, add it in there. It's possible, I'm not a great listener.
